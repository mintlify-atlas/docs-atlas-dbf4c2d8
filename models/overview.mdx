whisper.cpp uses a custom binary format called `ggml` to load and run Whisper models in C/C++ without Python dependencies.

## The ggml format

The original OpenAI PyTorch models are converted to a custom binary format that packs everything needed into a single file:

- **Model parameters** — architecture configuration (layers, heads, dimensions)
- **Mel filters** — filterbank coefficients for audio preprocessing
- **Vocabulary** — token-to-text mapping for decoding
- **Weights** — the trained model tensors in F16 or F32 precision

This format makes it straightforward to load models in C/C++ with zero external dependencies and zero runtime memory allocations.

## Model variants

### Multilingual vs. English-only

Models **without** a `.en` suffix are multilingual and can transcribe and translate audio in many languages. Models **with** a `.en` suffix (for example, `base.en`) are fine-tuned for English-only audio and are generally faster and more accurate for English content.

### Quantized models (`-q5_0`)

Models ending in `-q5_0` are integer-quantized variants that require significantly less disk space and memory. They can also run faster on certain hardware. See [Quantization](/models/quantization) for details on how to create quantized models yourself.

### tinydiarize models (`-tdrz`)

Models ending in `-tdrz` (for example, `small.en-tdrz`) support local speaker diarization. During transcription, the model marks speaker turns inline with `[SPEAKER_TURN]` tokens. This feature requires passing the `-tdrz` flag to `whisper-cli`.

### Fine-tuned and distilled models

whisper.cpp supports models fine-tuned using the Hugging Face Transformers implementation of Whisper, as well as distilled models from the [distil-whisper](https://huggingface.co/distil-whisper) project. These models are in a slightly different format and must be converted before use. See [Converting models](/models/conversion) for instructions.

<Note>
Models prefixed with `for-tests-` in the repository are empty placeholder files used by CI. They do not contain any weights and cannot be used for inference.
</Note>
