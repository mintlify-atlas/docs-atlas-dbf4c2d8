title: "Quickstart"
description: "Clone the repo, build whisper.cpp, and transcribe your first audio file."
---

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/ggml-org/whisper.cpp.git
    cd whisper.cpp
    ```
  </Step>

  <Step title="Build with CMake">
    ```bash
    cmake -B build
    cmake --build build -j --config Release
    ```

    The compiled binaries are placed in `./build/bin/`.
  </Step>

  <Step title="Download a model">
    Download one of the Whisper models converted to `ggml` format. The `base.en` model is a good starting point:

    ```bash
    sh ./models/download-ggml-model.sh base.en
    ```

    This saves the model to `models/ggml-base.en.bin`. To see all available models, run the script without arguments.

    <Note>
      Models are also available directly from [HuggingFace](https://huggingface.co/ggerganov/whisper.cpp) if you prefer to download them manually.
    </Note>
  </Step>

  <Step title="Transcribe an audio file">
    Run inference on the included sample audio file:

    ```bash
    ./build/bin/whisper-cli -m models/ggml-base.en.bin -f samples/jfk.wav
    ```

    You should see timestamped output like this:

    ```text
    [00:00:00.000 --> 00:00:11.000]   And so my fellow Americans, ask not what your country can do for you, ask what you can do for your country.
    ```

    <Tip>
      Run `./build/bin/whisper-cli --help` to see all available options, including output formats, language selection, and performance tuning flags.
    </Tip>
  </Step>
</Steps>

## Audio format requirements

whisper-cli requires **16-bit WAV** audio files sampled at 16 kHz. If your audio is in a different format, convert it with `ffmpeg` before transcribing:

```bash
ffmpeg -i input.mp3 -ar 16000 -ac 1 -c:a pcm_s16le output.wav
```

<Note>
  When built with the `WHISPER_FFMPEG` flag on Linux, whisper-cli can accept additional formats (Opus, AAC, and others) directly. See the [Installation](/installation) page for details.
</Note>

## Quick demo with Make

For a quick demo without manually downloading models, run:

```bash
make base.en
```

This downloads the `base.en` model and runs transcription on all `.wav` samples in the `samples/` folder.
