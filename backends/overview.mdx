whisper.cpp runs on CPU by default with no extra dependencies. For faster inference, enable a hardware acceleration backend at build time using a CMake flag.

## Backend comparison

| Backend | Hardware | CMake flag | Notes |
|---|---|---|---|
| Apple Metal | Apple Silicon (M-series) GPUs | `-DGGML_METAL=1` (default on macOS) | Full GPU inference via Metal |
| Core ML | Apple Neural Engine (ANE) | `-DWHISPER_COREML=1` | Encoder only; requires model conversion |
| NVIDIA CUDA | NVIDIA GPUs | `-DGGML_CUDA=1` | cuBLAS + custom CUDA kernels |
| Vulkan | Any Vulkan-capable GPU | `-DGGML_VULKAN=1` | Cross-vendor; requires Vulkan driver |
| OpenVINO | Intel CPUs and Intel GPUs | `-DWHISPER_OPENVINO=1` | Encoder only; requires model conversion |
| OpenBLAS | Any CPU | `-DGGML_BLAS=1` | CPU-only; good for POWER architectures |
| CANN | Ascend NPUs (Atlas series) | `-DGGML_CANN=1` | Huawei Ascend NPU via CANN toolkit |
| Moore Threads MUSA | Moore Threads GPUs | `-DGGML_MUSA=1` | muBLAS + custom MUSA kernels |

## CPU-only fallback

If no acceleration flag is set, whisper.cpp builds with CPU-only inference. This works on every supported platform and requires no extra dependencies:

```bash
cmake -B build
cmake --build build -j --config Release
```

CPU inference uses AVX/AVX2 intrinsics on x86, ARM NEON on Apple Silicon and ARM, and VSX intrinsics on POWER9/10 automatically when the compiler detects the target architecture.
