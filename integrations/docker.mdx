---
title: "Docker"
description: "Run whisper.cpp in a container using the official Docker images."
---

whisper.cpp provides official Docker images hosted on the GitHub Container Registry. The images include the `whisper-cli`, `whisper-server`, and `whisper-bench` executables along with `curl` and `ffmpeg`.

## Prerequisites

- Docker must be installed and running on your system.
- Create a local directory to store models and intermediate files (for example, `/whisper/models`). Models can be several gigabytes, so a persistent volume is recommended.

## Available images

| Image | Description | Platforms |
|-------|-------------|-----------|
| `ghcr.io/ggml-org/whisper.cpp:main` | Main image with `whisper-cli`, `curl`, and `ffmpeg` | `linux/amd64`, `linux/arm64` |
| `ghcr.io/ggml-org/whisper.cpp:main-cuda` | Same as `main`, compiled with CUDA support | `linux/amd64` |
| `ghcr.io/ggml-org/whisper.cpp:main-musa` | Same as `main`, compiled with MUSA support | `linux/amd64` |
| `ghcr.io/ggml-org/whisper.cpp:main-vulkan` | Same as `main`, compiled with Vulkan support | `linux/amd64` |

## Usage

### Download a model

Download a model and persist it in a local folder so it is available across container runs:

```shell
docker run -it --rm \
  -v path/to/models:/models \
  ghcr.io/ggml-org/whisper.cpp:main \
  "./models/download-ggml-model.sh base /models"
```

### Transcribe an audio file

Mount both a models directory and an audio directory, then run `whisper-cli`:

```shell
docker run -it --rm \
  -v path/to/models:/models \
  -v path/to/audios:/audios \
  ghcr.io/ggml-org/whisper.cpp:main \
  "whisper-cli -m /models/ggml-base.bin -f /audios/jfk.wav"
```

You can also transcribe the built-in sample files without mounting a separate audio directory:

```shell
docker run -it --rm \
  -v path/to/models:/models \
  ghcr.io/ggml-org/whisper.cpp:main \
  "whisper-cli -m /models/ggml-base.bin -f ./samples/jfk.wav"
```

### Run whisper-server

Start the HTTP transcription server and expose port 8080:

```shell
docker run -it --rm -p "8080:8080" \
  -v path/to/models:/models \
  ghcr.io/ggml-org/whisper.cpp:main \
  "whisper-server --host 127.0.0.1 -m /models/ggml-base.bin"
```

### Run whisper-bench

Benchmark encoder performance on a model using 4 threads:

```shell
docker run -it --rm \
  -v path/to/models:/models \
  ghcr.io/ggml-org/whisper.cpp:main \
  "whisper-bench -m /models/ggml-small.en.bin -t 4"
```

## GPU images

To use the CUDA image with an NVIDIA GPU, pass the `--gpus` flag:

```shell
docker run -it --rm --gpus all \
  -v path/to/models:/models \
  -v path/to/audios:/audios \
  ghcr.io/ggml-org/whisper.cpp:main-cuda \
  "whisper-cli -m /models/ggml-base.bin -f /audios/audio.wav"
```

<Warning>
  The `main-cuda` image requires the NVIDIA Container Toolkit to be installed on the host. See the [NVIDIA documentation](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html) for setup instructions.
</Warning>

<Note>
  Audio files must be 16-bit WAV format at 16 kHz. Use `ffmpeg` to convert other formats before transcribing:

  ```shell
  ffmpeg -i input.mp3 -ar 16000 -ac 1 -c:a pcm_s16le output.wav
  ```
</Note>
